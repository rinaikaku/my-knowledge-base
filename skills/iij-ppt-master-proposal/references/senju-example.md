# Senju Example Pattern

Use this reference when the user asks for a deck similar to the Senju/IIJ operation support and M365 migration proposal.

## Source Role Mapping

- Operation definition document: current contracted maintenance scope.
- PC operation/support quotation: candidate scope for PC support completion; omit all prices.
- Mail migration prerequisite note: migration assumptions, preparation items, schedule, and cautions.
- M365 quotation: candidate M365 service/work scope; omit all prices.
- IIJ company introduction deck or website screenshot: visual DNA only.

## Recommended 12-Page Structure

1. Cover: proposal title, customer, IIJ entity, date.
2. Positioning: two additional consideration domains.
3. Current operation scope: existing windows, support structure, reports, operational tasks.
4. PC support gap: user PC questions, triage, endpoint settings, on-site need.
5. PC support scope: sites/devices, remote support, failure handling, on-site response.
6. PC operation flow: intake, triage, remote response, on-site response, reporting.
7. Mail migration background: Alibaba Mail to M365 China as standardization step.
8. M365 scope: users, mailboxes, tenant setup, accounts, licenses, PC client settings.
9. Preparation information: domain admin, current mail admin, user list, shared mail, individual settings.
10. Cutover cautions: mail data migration excluded by default; Foxmail remains reference; PST can be mounted in Outlook when available; PC requirements.
11. Schedule: M365 environment prep, site-by-site cutover, endpoint settings, stabilization.
12. Next actions: scope, counts, special PCs, data handling, site order, on-site dates.

## Visual Pattern

- Use a black top strip or red active strip, echoing IIJ website navigation.
- Use red as the main accent, black for strong header blocks, gray for separation.
- Favor tables, two-column structures, process arrows, and numbered lists.
- Avoid blue-primary themes unless the user explicitly overrides.

## Copy Rules

- Write assertion titles in Japanese business style.
- Use "想定", "確認", "ご検討", and "前提" when facts are not finalized.
- Do not mention supplier quotes or internal cost basis in visible content.
- Keep titles to one line when possible.
