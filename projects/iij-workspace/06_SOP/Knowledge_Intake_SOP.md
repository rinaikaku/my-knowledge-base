# 知识录入标准作业流程

## 目的

当新的客户材料、提案文件、会议记录、邮件、产品说明或合规参考资料需要转换为可复用知识库条目时，使用本 SOP。

## 标准请求格式

```text
客户：[客户名称]
内容类别：客户 / 产品 / FAQ / 提案 / 邮件 / SOP / AI / 合规 / 不确定
参考资料：附件、路径、会议摘要或原始文字片段
特殊要求：请分类更新知识库；不要保存敏感价格、账号密码、内部审批信息或私有网络细节。
```

## 录入原则

- 用户提供的材料、项目事实、参数、报价、时间节点、结论、待确认事项和后续行动，默认均应完整记录到知识库。
- 区分客户专属知识和可复用知识。
- 一个文件只记录一个主要主题。
- 新建或更新知识条目时，默认使用中文。
- 仅在客户对外话术、来源原文、产品/技术术语、文件名或必须保持原文准确性的内容中保留日文或英文。
- 原材料中带有日期或具体时间时，新录入内容应尽量保留对应日期和时间；过往已录入内容暂不强制回溯补齐。
- 优先沉淀稳定的提案逻辑、问题模式、方案映射、FAQ 和可复用话术。
- 每项任务结束时，即使用户没有明确要求归档，也默认判断新信息是否应同步到知识库。
- 客户联系人姓名、公司邮箱、职务、部门、沟通角色等商务必要个人信息，可以保存在对应客户专属文件的“联系人信息”中。
- 个人手机号、私人邮箱、身份证件、家庭信息、健康信息等非业务必要个人信息，默认不入库；除非用户明确要求且确认有业务必要性。
- 不要保存密码、API Key、私钥等秘密原文；仅记录其存在、保管责任和受控存放位置。
- 客户价格、折扣、项目参数和敏感谈判信息应完整保存在受控客户专属记录中，标记复用边界；不得写入通用知识或跨客户复用材料。
- 如果来源包含详细拓扑或 IP 信息，只记录可复用架构模式，不把实施细节放入通用知识库。
- 如果法律、合规、产品规格、价格或监管状态可能变化，标记为“对外使用前需确认最新来源”。
- 如果来源包含日期、截止时间、会议候选时间、报价有效期、客户预计回复时间、续约日期、项目里程碑或其他时效性事项，需要在最终回复中作为跟进提醒列出。

## 分类规则

| 来源内容 | 目标区域 | 处理规则 |
|---|---|---|
| 客户背景、当前问题、决策人、联系人、现场情况 | `05_Customer` | 每个客户建立或更新一个客户文件；商务联系人信息仅保存在客户专属文件 |
| 产品或服务说明 | `01_Product` | 按可复用方案建立或更新产品/分类文件 |
| 客户问题、异议、重复回答 | `02_FAQ` | 增加简洁问答和异议处理 |
| 提案主线、路线图、价值逻辑、可复用话术 | `03_Proposal` | 建立案例提案逻辑或通用提案模式 |
| 邮件话术 | `04_Email` | 只保存可复用模板，不保存一次性机密内容 |
| 可重复执行的工作流程 | `06_SOP` | 新增或更新 SOP 文件 |
| AI 提示词或审查流程 | `07_AI` | 增加提示词模式和审查规则 |
| 法律、标准、认证、合规表达 | `08_Compliance` | 保留来源含义并增加可追溯性 |
| 导航变化 | `99_Index` | 新增可复用文件时更新索引 |

## 处理流程

1. 确认来源资料可以读取。
2. 从来源中提取文字和有用结构。
3. 识别客户专属事实、可复用提案逻辑、产品分类、FAQ、合规要点和敏感细节。
4. 将敏感细节完整记录在受控客户专属文件中，并排除在通用文件之外。
5. 没有合适文件时，新建 Markdown 文件。
6. 已有主题文件时，更新现有文件。
7. 必要时更新 `99_Index/Main_Index.md` 和 `99_Index/Product_Index.md`。
8. 验证 Markdown 链接。
9. 汇总变更文件、验证结果、假设和剩余风险。

## 输出检查清单

- 提到客户名称时，已创建或更新客户文件。
- 方案领域可复用时，已创建或更新产品/分类文件。
- 来源包含销售主线时，已沉淀提案逻辑。
- 出现客户问题或异议时，已更新 FAQ。
- 涉及监管、评价体系、认证或法律表达时，已更新合规文件。
- 索引文件已按需要更新。
- 敏感细节已排除，或已明确限制在客户专属记录中。
- 所有来源中的项目事实、参数、报价、时间节点、结论、待确认事项和后续行动均已记录，或明确说明无法提取的原因。
- 需要最新来源确认的风险已标记。
- 存在时效性事项时，已在最终回复中列出跟进提醒。

## 后续使用示例

同一客户的后续请求可以这样写：

```text
继续更新[客户名称]案件：根据今天的日文邮件补充项目进展和后续行动。
```

```text
继续更新[客户名称]案件：客户反馈对方案价格有疑问，请沉淀到 FAQ 和阶段说明。
```

其他客户的新请求可以这样写：

```text
新客户：[客户名称]；行业：[行业]；关注产品：[产品/方案]。请参考已有相似案例，生成提案大纲和需要确认的问题。
```

## 对外交付前复核点

- 确认当前产品规格和支持范围。
- 确认法律、法规、认证和评价体系的官方最新状态。
- 确认报价、范围、用户数、设备数、站点数和支持时间。
- 与客户负责人或交付团队确认客户专属假设。
