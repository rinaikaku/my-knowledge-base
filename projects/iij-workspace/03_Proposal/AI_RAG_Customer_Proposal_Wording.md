# AI 与 RAG 客户提案话术

## 目的

本文件用于沉淀企业 AI、RAG、API 集成、私有化部署和 AI 治理相关的客户提案话术。中文段落用于中国侧客户沟通，日文段落用于日本总部对齐或日文提案附件。

## 提案信息映射

| 客户议题 | 核心信息 | 提案强调点 |
|---|---|---|
| 企业 AI 导入 | AI 应被设计为业务和系统架构，而不只是模型使用。 | 先从业务流程出发，再定义知识、集成、安全和运营设计。 |
| RAG | RAG 在回答时连接模型和已批准的企业知识。 | 适用于内部规则、产品文档、客户资料、FAQ 和持续更新的运营知识。 |
| API 集成 | API 集成让既有系统可以自动调用 AI 能力。 | 应定位为应用集成、权限控制、监控和成本管理。 |
| 私有化部署 | 私有化部署可降低部分外部暴露，但不会免除治理工作。 | 仍需要身份、网络隔离、审计、补丁、访问控制和运营规则。 |
| AI Agent | Agent 可组合推理、检索、工具调用和流程动作。 | 只有在审批边界、日志和异常处理明确后才建议使用。 |
| 幻觉控制 | 模型输出可能看似合理但缺乏依据。 | 重要场景需结合 RAG、引用、结构化输出、人工复核和审计日志。 |

## 中文提案话术

### 执行摘要

企业 AI 的价值不只是调用大模型，而是将模型能力、企业知识、既有系统和安全治理组合成可运营的业务能力。建议先选择低风险、高复用的辅助型场景，通过 RAG 和 API 集成验证知识检索质量、答案可追溯性、权限控制和运营成本，再逐步扩展到更复杂的流程自动化。

### RAG 说明

RAG 可以理解为“先查企业已批准知识，再让模型基于查到的内容回答”。它不等同于重新训练大模型，更适合规则、产品资料、FAQ、客户材料等经常变化的企业知识场景。关键控制点包括知识源权限、检索质量、引用可追溯性、版本更新和人工复核。

### API 集成说明

API 集成的目的，是让现有业务系统在合适的流程节点调用 AI 能力，例如资料检索、摘要生成、问答、分类、检查或草稿生成。提案中应同时说明调用权限、日志、成本、失败处理和人工确认机制，避免把 AI 集成理解为简单接口调用。

### 私有化部署说明

私有化部署可以降低部分外部暴露，但并不自动解决数据安全和合规问题。仍需要设计身份认证、访问控制、网络隔离、数据脱敏、审计日志、补丁管理、备份恢复和运营责任。客户应把私有化部署视为整体治理方案中的一个组成部分。

### 推荐第一步

建议从内部 FAQ、产品资料检索、提案材料检索、销售支持或知识库录入等低风险场景开始。第一阶段重点验证检索准确率、来源可追溯性、用户采用率、节省工时、延迟、单次交互成本和人工升级比例。

## 日文提案话术

### 执行摘要

企業向け AI の価値は、大規模言語モデルを単体で利用することではなく、モデル機能、社内ナレッジ、既存システム、セキュリティガバナンスを組み合わせ、継続的に運用できる業務能力として設計する点にあります。まずは低リスクで再利用性の高い支援型業務から開始し、RAG と API 連携を通じて検索品質、回答根拠、権限制御、運用コストを検証することを推奨します。

### RAG 说明

RAG は、回答時に承認済みの社内ナレッジを検索し、その内容を根拠としてモデルに回答させる仕組みです。モデルを再学習する方式ではないため、規程、製品資料、FAQ、顧客資料など、更新頻度の高い社内知識に適しています。

### API 集成说明

API 連携により、既存の業務システムから AI 機能を自動的に呼び出すことができます。提案時には、単なるインターフェース連携ではなく、権限制御、ログ、コスト管理、例外処理、人による確認プロセスを含めて設計する必要があります。

### 私有化部署说明

プライベート環境での導入は外部露出を一部低減できますが、それだけでデータ保護やコンプライアンスが完結するわけではありません。認証、アクセス制御、ネットワーク分離、データマスキング、監査ログ、パッチ管理、運用責任を合わせて設計する必要があります。

### 推荐第一步

初期段階では、社内 FAQ、製品資料検索、提案資料検索、営業支援、ナレッジベース登録など、低リスクな支援型業務から開始することを推奨します。

## 提案结构模板

1. 客户业务背景和 AI 使用目标。
2. 当前知识、系统和流程痛点。
3. 推荐的第一阶段 AI / RAG 场景。
4. 架构说明：用户、应用、知识库、模型、API、日志和权限。
5. 安全与治理：数据边界、访问控制、审计、人工复核。
6. 阶段计划：PoC、试运行、正式导入、扩大适用范围。
7. 需要客户确认的事项。

## 客户确认清单

- 目标业务流程是什么？
- 哪些知识源可以被 AI 使用？
- 谁有权限查看哪些知识？
- 是否允许调用外部 LLM API？
- 是否需要私有化部署或专属环境？
- 回答是否需要显示引用来源？
- 哪些场景必须保留人工确认？
- 日志、保留期限和审计要求是什么？
- 成本、性能和响应时间目标是什么？
