# NTT Innovative Devices SMX 服务恢复策略

## 2026-07-13 最新状态

- **来源事实**：客户反馈 DNS 服务商已于 5 月确认无异常；邮件当时已可正常发送和接收。
- **来源事实**：客户已开始使用 Microsoft Exchange，并将联络域名切换为 `ntt-devices-asia.com`。
- **未闭环事项**：大附件接收仍可能困难；如复发，应收集发生时间、附件大小、发件/收件地址和错误信息后再调查。
- **提案影响**：原 SMX 常规收发异常不再作为正在持续的故障表述。香港或云侧稳定化方案应作为待重新确认的中长期韧性选项，而非基于旧事件直接承诺的既定迁移项目。
- **待确认**：Exchange 启用后，SMX 是否仍位于邮件路径及其实际支持范围。

## 客户背景

NTT Innovative Devices 深圳在五一假期前后经历了 SMX 邮件服务中断。该问题影响了业务连续性，并削弱了客户对中日支持模式的信任。

## 当前问题

- 中国到日本侧 SMX 邮件服务的跨境访问容易受到 DNS 和网络政策波动影响。
- 客户感受到本地支持偏被动、偏合同边界，而不是以业务连续性为导向。
- 深圳本地相关方对第一阶段合规咨询和第二阶段安全加固持谨慎态度。
- 本地预算敏感，第一阶段如果由深圳侧直接付款会比较困难。

## 提案目标

通过技术稳定化、可见的中日联合支持，以及降低本地预算阻力的合同模式，恢复客户信任。将 SMX 事件作为第二阶段韧性投资的证据。

## 核心提案逻辑

1. 从客户业务痛点切入：邮件不可用时，无论正式服务边界如何，客户运营连续性都受到了影响。
2. 将根因解释为跨境稳定性问题，涉及日本侧 DNS 解析和中国网络政策波动。
3. 将香港临时租户作为更有韧性的香港/云侧邮件架构的概念验证。
4. 将第一阶段合规咨询重新定位为避免监管暂停、处罚和总部升级的风险保险。
5. 将第二阶段定位为长期恢复方案：邮件迁移、安全加固和韧性支持流程。
6. 通过日本对日本签约、中国侧本地执行和报告，降低预算阻力。

## 推荐方案

### 第一阶段：合规咨询

- 合同结构：日本总部与日本侧签约，中国团队为深圳执行本地访谈、风险识别和报告交付。
- 销售表达：合规不是文书工作，而是防止业务暂停和行政处罚风险的预防控制。
- 必要证据：对外使用前准备近期中国数据合规执法案例。

### 第二阶段：安全加固和邮件稳定化

- 在当前项目周期中，9 月前定义需求，以支持下一财年预算计划。
- 将正式邮件域名/服务迁移到香港或云侧架构，定位为成功临时绕行方案的长期延续。
- 将邮件稳定性与更广泛的安全加固打包，让客户看到路线图，而不是一次性修补。

### 服务模式改善

- 即使问题可能位于本地支持合同边界外，也要展示中日团队主动联合调查。
- 建立主动升级沟通模式：当前状态、调查负责人、日本侧联动、预计下次更新时间、临时方案。
- 在信任恢复前，将沉默的相关方视为高风险。
- 混合型邮件事件应拆成两条调查线：
  - 中国大陆到日本 Secure MX 网络路径稳定性。
  - 发件人/收件人域名 DNS 解析错误，例如 `Domain not found`。
- 如果受影响域名的 DNS 服务由客户向第三方签约，应明确说明 IIJ 可检查 SMX 侧状态和名称解析行为，而客户或 DNS 供应商必须检查 DNS 服务故障、响应延迟、设置变更和内部日志。

## 对客户的价值

- 降低跨境邮件访问中断复发风险。
- 让深圳和日本总部获得共同说明和路线图。
- 降低本地预算阻力，同时保留本地交付质量。
- 将负面事件转化为结构化合规和安全改善项目。

## 风险与假设

- 对外交付前，必须用技术日志确认准确技术根因。
- DNS 解析问题涉及的域名为 `ntt-devices-sz.com`。
- 涉及 GFW 的措辞应谨慎、事实化。
- 中国数据合规案例在使用前必须用最新官方或可靠来源刷新。
- 合同和分包结构需由商务和法务相关方确认后再展示。
- 9 月需求定义时间基于当前项目周期，应与客户预算计划再次确认。
