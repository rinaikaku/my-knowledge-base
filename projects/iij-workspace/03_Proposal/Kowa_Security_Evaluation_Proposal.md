# 兴和制药中国安全评价提案

## 客户背景

兴和制药中国被定位为在中国运营、并与医疗健康相关业务伙伴连接的医药客户。本提案将网络安全视为运营连续性问题，也视为供应链信任问题。

## 当前问题

- IT 基础尚未形成足够结构化的外部攻击防护能力。
- 终端、ID、资产、网络和监控控制需要分阶段加强。
- 安全事件可能影响品牌信任、医疗相关业务连续性，以及供应商/客户对其安全状态的认可。
- 客户需要同时满足中国本地合规要求，并对齐日本总部期待。

## 提案目标

将日本安全措施评价体系作为 IT 实施和成熟度可视化的参考框架，先建立相当于 Star 3 的实用基础，再以 Star 4 作为中期参考目标逐步提升环境。该表述不意味着中国法人必须正式采用 SCS。

## 核心提案逻辑

1. 说明为什么评价体系可作为参考：供应链攻击使安全状态成为交易要求，而不仅是内部 IT 问题。
2. 定位客户风险：医疗相关客户需要高度信任，停机或 IT 环境被滥用会带来声誉和业务影响。
3. 将 Star 等级作为内部实施路线图：Star 1-2 是基础，Star 3 是最低实施要求，Star 4 是标准目标，Star 5 是高级供应链状态。
4. 将路线图映射到具体方案领域：终端、ID、网络、云、邮件、Web、监控、评估、培训和合规。
5. 商业估算应与可复用知识分开呈现，因为具体价格属于客户敏感信息。

## 推荐路线图

### 第一步：Star 1 基础

- 替换老旧 PC 和不受支持的 OS 环境。
- 按需要对旧 PC 做数据擦除、物理销毁并出具证明。
- 建立终端和 ID 台账。

### 第二步：Star 2 入侵前提基础

- 视情况将 AD、文件服务器、管理系统等核心服务器迁移到 IIJ GIO China。
- 实施资产管理、终端管理、EDR、防火墙/UTP 和网络设备更新。
- 增加安全意识培训和运营支持。

### 第三步：Star 3 最低安全实施

- 增加 ZTNA + SWG、邮件安全网关、钓鱼邮件演练、安全评估和等保二级支持。
- 执行外部资产漏洞评估、内部资产漏洞评估和关键设备基线评估。

### 第四步：Star 4 标准目标状态

- 增加 SOC 监控和事件响应联动。
- 通过渗透测试验证入侵路径和整改有效性。
- 根据需要加强 XDR、CASB/SWG、DLP、MFA、ASM 和安全人才培养。

### 第五步：Star 5 高级状态

- 当客户需要更高级的一体化网络和安全能力时，考虑 SASE 和更广泛的供应链导向控制。

## 对客户的价值

- 提供可向总部、客户和供应商说明的清晰成熟度路线图。
- 降低跳板攻击风险和业务连续性风险。
- 将碎片化安全检查转换为统一的安全成熟度语言。
- 帮助客户按阶段确定投资优先级，而不是把安全控制看作孤立产品。

## 风险与假设

- 来源资料中将安全措施评价体系描述为拟议或发展中的框架；对外交付前需确认最新官方状态。
- 报价提交前，必须重新确认具体成本、详细拓扑和实施范围。
- 中国网络安全、个人信息保护和等保相关表达应在客户交付前复核。
- 由于中国业务目前没有专职 IT 负责人，在最终确定范围、优先级和执行责任前，应与日本总部 IT 一起确认改善项目。
- SCS 是来自日本的框架，在本中国项目中只应作为 IT 实施参考。不要暗示中国法人必须导入或通过 SCS 认证。

## 可复用客户话术

```text
安全措施评价体系是来自日本的参考框架，并不表示中国法人必须正式导入该制度。本项目中可将其作为整理 IT 实施优先级和安全对策等级的参考尺，先建立基础对策，再按阶段推进必要的安全控制。
```

```text
在中国侧没有专职 IT 负责人的前提下，改善计划不应只依据本地运营实际，还需要结合日本总部 IT 部门的管理方针、安全基准和审批流程。因此，下一步建议安排中国侧相关人员与日本总部 IT 共同参加的沟通会议。
```
