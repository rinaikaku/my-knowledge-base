## 1. 目标与边界

基于用户提供的《IIJ 中国 DC 网络拓扑见直し相关信息提供资料》，提炼适用于中国站点访问 Microsoft 365 国际版的推荐使用方式，形成可用于项目立项、PoC 和客户沟通的技术方案。

报告不复述客户专属报价、设备明细、人员规模或排期；此类信息仅进入受限知识库索引。报告中的性能数字均标记为源材料中的历史观测值，不构成现网 SLA 或路由承诺。

## 3. 推荐架构

采用“PAC 精准分流 + M365 专用安全出口 + 上海至东京 IPLC + 日本侧 IIJ/DIA 至 Microsoft 网络”的路径：

1. PAC 依据 Microsoft 365 官方端点类别维护规则；不硬编码静态 IP。
2. M365 流量不经通用显式代理，进入专用 NGFW 安全出口后走 IPLC 至日本侧 DIA，与 Microsoft 网络互联。
3. 非 M365 互联网流量继续经企业代理/NGFW 后从中国本地 DIA 出口访问。
4. 保留日志、监控、跳板机 MFA、配置备份、漏洞扫描和基线加固等运营控制。
