## 当前问题

- IT 基础被描述为接近未充分建设状态，对外部攻击的防护能力有限。
- 安全薄弱可能形成跳板风险，即客户环境被攻击者利用后，进一步攻击医疗机构或其他业务伙伴。
- 如果发生安全事件，客户可能不仅被视为受害者，也可能被认为对下游损害负有管理责任。
- 本地运营需要更清晰的权限管理、运营规则、终端控制、网络控制、监控和合规对齐。

## 业务风险

- 安全事件导致品牌和社会信任受损。
- 医疗相关业务存在中断风险。
- 如果供应商或客户开始要求可见的安全证明，交易连续性可能受影响。
- 中国网络安全和个人信息保护要求下可能存在法律和行政风险。
- 可能的所有权/治理变化会影响审批路径、预算时间、IT 外包连续性和日本总部参与方式。该事项目前未验证，必须谨慎处理。

- 使用日本提出的安全措施评价体系作为共同参照尺，帮助可视化当前成熟度并说明改善优先级。

## 现有合同与服务边界

- 当前签约内容 1：每周一天 IT 驻场对应 + 5x8 Helpdesk 远程对应。
- 驻场业务范围：现场 IT 问题对应 / 问题发现；IT 安全强化计划策定。
- Helpdesk / SI 运维保守支持范围：对象为 PC、服务器、网络设备；不包含业务应用程序等。
- 历史签约内容 2：在上述驻场与 Helpdesk 服务签约前，FW 和服务器维护支援曾作为单独合同提供。
- FW / 服务器单独支援内容包括：配置变更、停电对应（上门）、远程固件升级等。
- 合同周期补充：驻场 / Helpdesk 服务与 FW / 服务器单独维护服务的签约周期不完全重合。具体起止日期、续约条件及可能的空档期需以合同正文确认。
- 预判客户可能在下次会面提出：希望用当前每周驻场 + 5x8 Helpdesk 体制覆盖原本单独签约的 FW / 服务器维护支援。
- 服务能力补充：当前驻场 / 5x8 Helpdesk 并非仅限于一次对应。遇到需要专业判断或处置的事项，可升级并协调工程师支援。
- 风险管理判断：不建议简单用当前驻场 / Helpdesk 合同吸收 FW / 服务器单独支援。关键差异不在于“是否有工程师能力”，而在于是否已为高风险基础设施作业预先定义并承诺服务条件；同时，两份服务的合同周期不重合，不能仅凭日常运作习惯跨越合同有效期默认提供维护。FW / 服务器的配置变更、固件升级和停电现场对应，需要明确作业前风险评估、审批、维护窗口、工程师资源预留、备份与回滚、现场到场、厂商协调、故障恢复时限及责任边界。现有服务可作为问题受理、现场协调和按需升级入口，但不应被默认解释为已包含上述计划性维护与业务连续性保障责任。建议延续单独支援体制；如需整合，必须以正式变更方式统一服务期间，并重新定义范围、SLA、变更流程、响应与恢复责任及费用。
