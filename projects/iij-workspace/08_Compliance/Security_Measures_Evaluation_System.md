# 安全措施评价体系

## 目的

本文件用于沉淀日本拟推进的“安全措施评价体系”相关说明和提案话术，重点用于供应链网络安全状态可视化。

面向中国法人的提案中，只应将该体系作为来自日本的 IT 实施和成熟度可视化参考。除非客户或总部明确要求，否则不要暗示中国法人必须正式采用 SCS。

## 来源文本或要求

- 来源资料引用了 METI 关于供应链强化安全措施评价体系的中间报告。
- 官方参考：`https://www.meti.go.jp/shingikai/mono_info_service/sangyo_cyber/wg_seido/wg_supply_chain/pdf/20250414_2.pdf`
- 该体系被定位为订货企业和供应商之间检查、说明网络安全措施的共同参照尺。
- 来源资料说明，体系主要审查对象为 Star 3 至 Star 5，SECURITY ACTION 覆盖类似 Star 1 和 Star 2 的自我声明。

## 等级概要

| 等级 | 定位 | 评价方式 | 提案用途 |
|---|---|---|---|
| Star 1 | SECURITY ACTION 声明和基础终端/ID 意识 | 自我声明 | 入门级意识和基础卫生 |
| Star 2 | 基础方针和以入侵为前提的基础能力 | 自我声明 | 终端、ID、网络、防火墙、EDR、教育和资产管理基础 |
| Star 3 / Basic | 应实施的最低安全控制 | 自我评估 | 基础薄弱客户的实际第一目标 |
| Star 4 / Standard | 标准目标安全控制 | 第三方评价 | 处理重要客户信息或关键业务运营客户的中期建议目标 |
| Star 5 | 高级目标状态 | 第三方评价 | 高级供应链和高成熟度状态 |

## 等级判断逻辑

- Star 3 和 Star 4 应同时从业务和系统视角判断。
- 业务判断包括是否处理订货方的重要机密信息，以及业务中断是否会造成不可接受的延误。
- 系统判断包括供应商环境是否可能访问订货方内部系统。
- 如果存在重要信息、不可接受的延误或内部访问风险，应考虑将 Star 4 作为目标状态。

## 客户对外话术

```text
安全措施评价体系是用于将不同企业之间的安全确认项目可视化的共同参照尺。它不是单纯为了取得认证，而是帮助订货方和供应商双方说明安全措施水平、识别供应链风险，并决定后续改善优先级的实务框架。
```

## 提案备注

- 在确认最新官方状态前，不要将该体系描述为已完全定稿。
- 除非项目范围已确认，不要承诺取得认证、第三方评价结果或监管认可。
- 面向中国客户时，应将其解释为日本供应链期待，同时另行说明中国网络安全、个人信息保护和等保要求。
- 用于中国法人项目时，应定位为实施参考，而不是中国侧强制合规要求。

## 相关方案领域

- 终端盘点和操作系统更新。
- AD / ID 管理、SSO、MFA、IGA 和 ITDR。
- 防火墙、IPS/IDS、WAF、DDoS、SASE、ZTNA、SWG 和 CASB。
- EPP、EDR、MDR、XDR、SOC、SIEM、UEBA 和威胁狩猎。
- 漏洞评估、ASM、CTEM、SBOM 和渗透测试。
- 备份、恢复演练、云备份和抗勒索数据保护。
- 第三方风险评估和供应链安全评估。
