## 核心结论

在中国开展 IT 与数据处理业务时，安全治理不能只停留在网络防护产品部署。企业需要同时覆盖资产与网络可视化、账号与终端控制、数据分类分级、个人信息治理、日志审计、漏洞和事件响应、供应商管理，以及按业务系统判断的等保和数据出境事项。

## 合规框架

资料将以下事项视为中国安全与数据治理的主要框架：

- 《网络安全法》：网络运营安全义务、网络安全等级保护等要求。
- 《数据安全法》：数据处理活动、数据安全管理与风险治理。
- 《个人信息保护法》：个人信息处理的合法性、告知同意、个人权利与安全管理要求。
- 网络安全等级保护 2.0（等保）：按系统边界和保护等级实施技术与管理措施。
- 数据出境相关制度：应先识别数据类型、处理目的、流向、接收方、存储/备份/运维路径，再判断适用要求。

## 企业治理主线

1. 建立管理层负责的安全方针、组织和预算机制。
2. 梳理系统、资产、账号、数据、供应商和跨境数据流。
3. 对数据进行分类分级，明确个人信息、敏感个人信息、重要数据和业务秘密的处理边界。
4. 建立身份与访问控制、终端管理、网络分区、漏洞管理、日志监控、备份恢复和事件响应控制。
5. 通过培训、演练、审计和持续改进使制度真正落地。
6. 对面向公众、承载重要业务或具有特定影响的系统，按最新主管机关和测评机构要求确认等保路径。

## 对外说明的安全边界

- 不能仅凭云服务商或平台自身的认证、等保等级或安全能力，推定客户业务系统已满足全部义务。
- 数据位于中国、备份到海外、远程运维访问、日志同步和灾备恢复均可能构成需评估的数据流。
- 等保定级、备案、测评和主管机关受理均应由法定责任主体、指定机构及所在地最新口径确认。
- 对外引用法规、处罚案例或统计数字前，应回到最新官方来源核验，不直接引用本资料中的历史统计。

## 信息安全管理实践

参考《中小企业信息安全对策指南》，可采用以下持续改进闭环：

1. 由经营层明确安全责任、方针和资源投入。
2. 识别信息资产、威胁、脆弱性、影响和发生可能性，形成风险分析。
3. 制定适合本企业的安全政策、控制标准和实施流程，而不是仅套用模板。
4. 落实基础控制：账号与密码、终端和软件更新、恶意软件防护、备份、访问权限及外部委托管理。
5. 以问卷、访谈、文档复核和现场观察检查执行情况。
6. 根据业务变化、事件、审计结果和控制缺口持续修订政策与措施。

## 与客户项目的映射

| 客户问题 | 建议切入 |
|---|---|
| 不清楚中国 IT 现状 | 先做资产、网络、漏洞、日志和生命周期调查。 |
| 担心数据出境 | 建立数据清单和数据流图，确认数据类型、系统边界、存储、备份和运维路径。 |
| 需要等保 | 先确认系统边界、业务影响、现有控制和所在地要求，再制定整改与申报路径。 |
| 制度有但执行弱 | 结合员工培训、权限审计、日志抽查、演练和供应商管理验证执行。 |
