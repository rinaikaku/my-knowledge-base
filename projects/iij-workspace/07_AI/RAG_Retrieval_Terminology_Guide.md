# RAG 与检索术语指南

## 来源

- 来源：用户在对话中提出的 RAG、向量搜索、Text2SQL、VQA、Webhook、Embedding、Chunking、Rerank、HyDE、Agent、Fine-tuning、Graph RAG 等术语说明需求。
- 录入日期：2026-07-07。
- 分类：RAG、检索、企业 AI、术语解释。

## 概要

RAG 是 Retrieval-Augmented Generation，即“检索增强生成”。简单说，就是先从企业批准的知识库中找到相关资料，再让模型基于这些资料回答问题。它适合需要引用来源、知识经常变化、不能把答案完全交给模型记忆的企业场景。

## 核心术语

| 术语 | 中文解释 | 客户说明重点 |
|---|---|---|
| RAG | 检索增强生成，先查资料再回答 | 不是训练模型，而是把当前问题连接到企业知识 |
| 向量搜索 | 将文本转换为向量后按语义相似度检索 | 适合文档、邮件、手册、FAQ |
| Text2SQL | 将自然语言问题转换成 SQL 查询 | 适合结构化数据库和报表问题 |
| VQA / DocVQA | 针对图片、扫描件、发票、截图或图表进行问答 | 适合图片或版式文档 |
| Webhook | 一个系统在事件发生时通知另一个系统的机制 | 适合流程触发和系统联动 |
| Embedding | 把文本转换成模型可比较的向量表示 | 是语义检索的基础 |
| Chunking | 将长文档切分成小段 | 影响检索质量和上下文完整性 |
| Rerank | 对初步检索结果重新排序 | 用于提高最相关内容的排序 |
| HyDE | 先生成假设性答案或文档，再用于检索 | 适合用户问题很短或表达模糊的场景 |
| Agent | 可组合检索、推理、工具调用和动作的 AI 系统 | 必须控制权限、日志和异常处理 |
| Fine-tuning | 用训练数据调整模型行为 | 与 RAG 不同，成本和治理要求更高 |
| Graph RAG | 将图谱关系与检索生成结合 | 适合组织、资产、系统、政策之间关系重要的场景 |

## 向量搜索与关键词搜索

关键词搜索依赖字面匹配；向量搜索关注语义相似度。客户如果问“为什么不用普通搜索”，可以这样解释：

```text
普通搜索更擅长查找包含同一关键词的内容。向量搜索更适合处理表达不同但含义相近的问题，例如客户用不同措辞描述同一产品、故障或制度要求时，系统仍可能找到相关资料。
```

## Text2SQL 使用边界

Text2SQL 适合需要从结构化数据中计算答案的场景，例如销售额、库存、订单、设备数量或告警统计。

注意事项：

- 需要清晰的数据表结构和字段含义。
- 需要控制用户能查询哪些表和字段。
- SQL 执行应有只读限制和审计。
- 重要报表结果需要校验。

## Chunking 设计

文档切分过小会丢失上下文，切分过大则可能引入无关信息并增加成本。

常见做法：

- 按标题、段落、条款或表格边界切分。
- 保留来源文件、页码、章节和更新时间。
- 对关键文档保留引用信息。
- 根据实际问答效果调整切分大小。

## Rerank 作用

Rerank 是对初步检索结果再次排序，让最相关的片段排在前面。它适合在文档多、相似内容多、初次检索噪音较高时使用。

## HyDE 作用

HyDE 会先根据用户问题生成一个假设性答案或假设性文档，再用它去检索。它适合用户问题过短、表达含糊或缺少关键词的场景，但也需要防止假设内容引入偏差。

## Graph RAG 适用场景

当问题不只是“哪段文字相关”，而是“哪些实体之间有什么关系”时，可以考虑 Graph RAG。

典型场景：

- 公司、部门、联系人、项目之间的关系。
- 系统、资产、漏洞、控制措施之间的关系。
- 合同、政策、条款、责任主体之间的关系。
- 客户、产品、提案、FAQ 之间的复用关系。

## RAG 与 Fine-tuning 的区别

| 维度 | RAG | Fine-tuning |
|---|---|---|
| 目标 | 让模型基于当前检索到的知识回答 | 调整模型行为或表达风格 |
| 更新方式 | 更新知识库或索引 | 准备训练数据并重新训练 |
| 适合内容 | 变化快、需引用来源的企业知识 | 稳定风格、特定任务模式 |
| 风险 | 权限、检索质量、上下文泄漏 | 训练数据治理、模型记忆、评估成本 |
| 初期建议 | 多数企业知识问答优先评估 | 有明确数据和目标后再考虑 |

## 客户常见解释

### RAG 是否会把资料训练进模型？

回答：RAG 本身是当前请求时的检索和上下文提供，不等同于训练模型。但如果使用外部 LLM API，供应商是否保存或复用输入输出，需要看合同条款、数据保留设置和审计证据。

### 企业知识经常变化怎么办？

回答：RAG 的优势是可以通过更新知识库、索引和权限来反映最新资料，而不需要每次重新训练模型。

### 为什么回答需要引用来源？

回答：引用来源可以让用户验证答案依据，也方便发现知识库内容是否过期、缺失或权限设置不当。

## 提案检查清单

- 知识源是否可靠并已批准。
- 是否包含客户敏感信息。
- 权限是否能继承或映射。
- 是否需要文档级、章节级或字段级引用。
- 是否需要 Text2SQL 或 Graph RAG。
- 是否需要多语言检索。
- 是否需要 Rerank。
- 是否需要人审。
- 是否记录检索和回答日志。
- 如何评估检索准确率和回答质量。
