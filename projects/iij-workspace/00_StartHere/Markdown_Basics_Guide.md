# Markdown 基础使用指南

## 来源

- 来源：用户在对话中提供的 Markdown 说明。
- 录入日期：2026-07-07。
- 分类：知识库使用、Markdown、写作格式。

## 概要

Markdown 是一种轻量级标记语言。简单说，它允许我们在纯文本中输入少量符号，例如 `#`、`**`、`-`、链接、表格和代码块，从而表达标题、加粗、列表、引用和结构化内容。

本知识库使用 Markdown，是因为它易读、易编辑、方便搜索，也更适合被 AI 助手读取、总结和复用。

## 为什么使用 Markdown

Markdown 解决的是日常写作中的结构化问题。相比 Word 或 WPS 中依赖鼠标、菜单和样式的操作，Markdown 可以直接用键盘输入结构。

主要优点：

- 纯文本文件几乎可以用任何编辑器打开。
- 标题和列表让内容更容易浏览。
- 链接可以连接相关知识页。
- 表格在源码和渲染后都容易阅读。
- 与二进制文档相比，更容易做版本差异对比。
- AI 工具可以稳定读取、总结和复用 Markdown 内容。

## 常用语法

| 目标效果 | Markdown 写法 | 呈现结果 |
|---|---|---|
| 一级标题 | `# 标题` | 大标题 |
| 二级标题 | `## 标题` | 章节标题 |
| 三级标题 | `### 标题` | 小节标题 |
| 加粗 | `**重要文字**` | 加粗文字 |
| 斜体 | `*强调文字*` | 斜体文字 |
| 无序列表 | `- 项目` | 项目符号列表 |
| 有序列表 | `1. 第一项` | 编号列表 |
| 链接 | `显示文字 + 路径或网址` | 可点击链接 |
| 图片 | `说明文字 + 图片路径或网址` | 嵌入图片 |
| 行内代码 | `` `代码` `` | 行内代码样式 |
| 代码块 | 三个反引号包住多行内容 | 多行代码块 |
| 表格 | 竖线加分隔行 | 结构化表格 |
| 引用 | `> 引用内容` | 引用块 |
| 分隔线 | `---` | 横向分隔线 |

## 表格示例

Markdown 源码：

```markdown
| 主题 | 向量搜索 | Text2SQL |
|---|---|---|
| 检索对象 | 文档 | 数据库表 |
| 方法 | 语义向量匹配 | SQL 查询生成 |
```

渲染后的含义：

| 主题 | 向量搜索 | Text2SQL |
|---|---|---|
| 检索对象 | 文档 | 数据库表 |
| 方法 | 语义向量匹配 | SQL 查询生成 |

## 代码块示例

当需要保留命令、SQL、提示词或示例的原始格式时，使用代码块。

````markdown
```sql
SELECT product_name, sales_amount
FROM sales
ORDER BY sales_amount DESC;
```
````

## Markdown 常见使用场景

Markdown 常用于：

- GitHub 的 `README.md` 文件。
- 技术博客。
- Obsidian 等笔记工具。
- 类 Notion 的写作流程。
- AI 助手回答。
- 本工作区这样的内部知识库。

## 本知识库如何使用 Markdown

本工作区使用 Markdown 来沉淀可复用知识：

- `#` 和 `##` 标题定义可搜索章节。
- 表格用于整理提案比较和客户说明。
- 链接用于连接主题页和索引页。
- 代码块用于保留提示词、命令、模板和示例。
- 纯文本格式方便 Codex 读取和更新。

## 实用写作规则

新增或编辑知识时：

- 一个文件只写一个主要主题。
- 文件开头使用清晰的 `#` 标题。
- 主要部分使用 `##` 分节。
- 比较多个选项时优先用表格。
- 可复用要点使用项目符号列表。
- 引用其他知识页时使用链接。
- 避免保存密码、密钥、私有网络细节或机密价格信息。

## 推荐工具

可以使用以下工具查看或编辑 Markdown：

- 普通文本编辑器。
- Visual Studio Code。
- Obsidian。
- Typora。

- Codex 工作区本身。

以下场景可以复用本条目：
