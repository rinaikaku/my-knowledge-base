# 网络基础设施与安全调查服务

## 条目信息

- 分类：产品
- 适用场景：网络架构不透明、资产与外部暴露面不清、漏洞与生命周期风险、性能问题及安全运维改进
- 来源：`東京エレクトロンネットワークセキュリティ調査_20250117c.pptx`
- 来源日期：2025-01-17
- 更新日期：2026-07-14
- 复用级别：通用
- 对外使用前复核：是；调查范围、授权、扫描窗口、数据处理、交付格式和报价需按项目确认。

## 一句话定位

通过对内部和外部资产、物理与逻辑网络、脆弱性、通信流、性能、生命周期及安全运维的调查，帮助客户形成可用于整改决策的网络安全现状基线。

## 调查目标

- 识别内部资产、外部暴露资产和攻击面。
- 还原机柜、线路、物理/逻辑拓扑、VLAN、IP 分配和访问控制。
- 识别网络与系统漏洞、EoL/EoS/EoVSS 生命周期风险。
- 分析应用访问路径、延迟、丢包、抖动、带宽和通信流。
- 复核防火墙、ACL、NAC、日志、监控、跳板机和运维管理情况。
- 输出整改建议，包括暴露面收敛、访问控制优化、漏洞修复、生命周期管理和运维改善。

## 调查方法

1. **资料与授权准备**：确认站点、联系人、网络范围、资产清单、线路、现有图纸、日志和变更记录；确认调查 PC、扫描、抓包和监控数据获取方式。
2. **资产发现**：从客户域名和公网 IP 识别外部暴露资产；在授权网段部署工具梳理内部资产。
3. **漏洞与配置复核**：对已确认资产进行漏洞扫描；结合设备登录、配置和版本信息检查安全基线、ACL、NAC、Firewall 策略及路由设计。
4. **性能与流量分析**：通过测试工具与镜像端口抓包，分析延迟、丢包、抖动、带宽占用、应用通信和业务路径。
5. **运维与审计调查**：复核监控、日志、时间同步、跳板机、权限、运维流程和变更记录。
6. **输出与改善**：形成拓扑图、资产与漏洞清单、生命周期、访问控制、性能、带宽和通信流报告，并提出整改优先级。

## 主要交付物

- 网络物理构成图和逻辑构成图。
- 内部/外部资产清单、开放端口与外部暴露面清单。
- 漏洞扫描结果和访问控制策略复核结果。
- 设备硬件/软件生命周期清单。
- 网络性能、带宽利用率和通信流分析。
- 安全监控、日志审计和运维管理现状及改善建议。

## 数据与安全边界

- 任何扫描、抓包、设备登录和日志读取均须取得书面授权，并确认窗口、影响控制和异常联络机制。
- 客户网络拓扑、IP、设备名、账号、日志和漏洞明细属于客户专属信息，不能写入通用方案或跨客户复用资料。
- 通用条目只保留方法、范围和交付物；具体资产与配置应保留在受限客户记录和源文件中。
