# AD / ID 管理

## 一句话定位

AD / ID 管理用于建立集中身份控制、认证和终端治理能力，是提升安全成熟度的基础。

## 适合客户

- 尚未标准化域或 ID 管理的客户。
- 多站点账号管理不一致的客户。
- 准备推进 SSO、MFA、MDM、IGA 或更高等级身份治理的客户。

## 核心价值

- 集中管理用户和设备。
- 支持认证与授权控制。
- 为终端策略、文件服务器访问控制和审计能力建立基础。

## 关键功能

- AD 部署和设计。
- ID 台账和定期复核。
- 认证与授权策略。
- 与文件服务器、终端管理和安全工具集成。

## 差异化

- 支撑后续 SSO、MFA、MDM、IGA、ITDR 等成熟度提升。
- 是从基础终端控制走向零信任访问前的重要基础。

## 常见客户问题

### 为什么要先做 AD / ID 管理？

回答：安全控制依赖于明确“谁在使用、使用哪台设备、允许访问什么”。如果缺少 ID 管理，后续 SSO、MFA、ZTNA 和审计都会更难稳定运行。

## 提案可用话术

- 整备 AD / ID 管理后，可以统一掌握用户、终端和访问权限，并为后续 SSO、MFA、零信任化建立认证基础。

## 技术备注

- 需要确认当前域结构、账号生命周期、组策略、文件服务器权限，以及中国侧/日本侧管理责任分工。
