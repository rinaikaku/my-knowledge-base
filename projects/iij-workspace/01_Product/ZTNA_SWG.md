# ZTNA + SWG

## 一句话定位

ZTNA + SWG 组合用于减少网络访问中的隐含信任，并为用户、设备和互联网访问建立更细粒度控制。

## 适合客户

- 需要安全远程访问，且不希望只依赖传统 VPN 的客户。
- 需要 URL 过滤、恶意软件防护和互联网访问控制的客户。
- 正在向 Star 3 或更高安全成熟度推进的客户。

## 核心价值

- 减少网络访问中的隐含信任。
- 控制用户和设备对应用的访问。
- 为访问互联网的流量增加安全 Web 访问和过滤能力。

## 关键功能

- Zero Trust Network Access。
- Secure Web Gateway。
- 基于用户/设备的策略控制。
- URL 过滤和 Web 安全。
- 根据厂商和范围提供终端部署和授权运营。

## 差异化

- 可作为从传统网络边界控制走向 SASE 的过渡。
- 当客户需要更聚焦的 Star 3 改善，而不是立即采用完整 SASE 时，可以优先提出。

## 常见客户问题

### ZTNA + SWG 是否等同于 SASE？

回答：不等同。ZTNA 和 SWG 是 SASE 的重要组成部分，但 SASE 通常还整合更广泛的网络和安全能力。ZTNA + SWG 可以作为完整 SASE 前的较小范围改善步骤。

## 提案可用话术

- ZTNA + SWG 可以先从用户访问和互联网访问控制切入，逐步减少传统 VPN 和边界防护的局限。

## 技术备注

- 设计前需确认身份提供方、终端操作系统、路由模式、代理模式、应用清单和中国访问需求。
