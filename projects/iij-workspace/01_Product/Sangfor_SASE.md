# Sangfor SASE

## 适用场景

- 面向中国境内分支、数据中心、云应用及远程用户的统一安全访问与网络连接管理。
- 适用于需要以零信任替代或补充传统 VPN、统一互联网访问控制、降低多客户端与多控制台运维复杂度的场景。
- 可与 SD-WAN、现有防火墙、代理、PAC 文件及既有终端安全体系并行或分阶段衔接；具体兼容性应以项目 POC 和设计验证为准。

## 核心组件与作用

| 组件 | 主要作用 | 典型使用方式 |
|---|---|---|
| ZTNA | 按用户、终端、应用及上下文实施最小权限访问，降低内部应用暴露面 | 通过连接器访问数据中心及多云私有应用；可对接 LDAP、ADFS/SAML、OIDC 等认证源，具体以环境验证为准 |
| SWG | 互联网访问认证、URL/应用分类控制、日志审计、威胁拦截与白黑名单管理 | 办公终端和分支的互联网访问控制；可按用户组及策略优先级管理 |
| XDLP（可选） | 数据外发控制、数据来源追踪、风险告警与溯源 | 结合文件来源、类型、名称、大小或内容等条件进行外发限制；应先验证业务规则与误报影响 |
| aES | EPP/EDR 一体化的终端保护与威胁检测响应 | 与 SASE 客户端策略协同；现有 EDR 多平台环境可先采用混合部署，避免一次性整合风险 |
| SD-WAN / GA | 分支组网、全球加速及网络与安全协同 | 依据分支规模、SLA、带宽和跨境需求选择 CPE、轻量接入或加速方案 |
| AIO 客户端 | 统一终端接入与策略承载 | 远程/移动用户使用；客户端推送、升级及终端兼容性需由客户侧管理工具或 AD 配合验证 |

## 方案设计要点

- 先完成应用、用户、终端、现有代理/PAC、出口流量及身份源梳理，再制定 IA（互联网访问）与 PA（私有应用访问）策略。
- IA 策略需单独确认本地互联网出口（LBO）、统一安全策略、特殊用户例外及需排流的应用。
- PA 策略应区分默认访问、重要应用的个别授权和后续待确认的例外访问。
- 对 SaaS 白名单或固定出口 IP 有要求时，应将 POP 选择、出口 IP、冗余和业务体验作为设计输入；不得将具体 IP 写入通用知识库。
- 建议采用 POC → 环境/需求调查 → 初步与详细设计 → 小范围预切换 → 分批切换 → 一个月稳定期 → 运维交接的阶段性推进方式。

## 提案与交付注意事项

- 应将产品能力、客户需求满足度和 POC 验证结果分开表述。产品资料中的覆盖、性能、市场份额、SLA 与安全效果数字均需在对外使用前由厂商最新资料或合同条款复核。
- 对“内容特征相近网站自动过滤”等细粒度需求，不应默认承诺满足；大金资料中该项被明确标为“不满足”，可改以现有 URL/应用分类能力和 POC 验证说明边界。
- XDLP 的功能、效果和告警指标应视为厂商资料中的能力主张，不得直接等同为客户环境中的保证结果。
- 跨境流量设计、现网 CATO/代理/防火墙变更、终端批量安装、业务集成测试及性能调优，必须在项目范围、责任边界和验收标准中单独确认。
